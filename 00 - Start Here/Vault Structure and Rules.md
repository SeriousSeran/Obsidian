# Vault Structure and Rules

This vault is organized as a folder-first Obsidian system.


## Rule 1 - Every folder needs an index

Each major folder should have a MOC, or Map of Content.

Examples:

- [[Medicine MOC]]
- [[Commerce MOC]]
- [[AI MOC]]
- [[Product Ideas MOC]]

## Rule 2 - Notes should be useful, not decorative

A good note should support one of these:

- Better clinical thinking
- Better exam answering
- Better money decisions
- Better product creation
- Better health and discipline
- Better AI/tool building

## Rule 3 - Clinical notes should be deidentified unless kept outside this repo

Clinical material should be transformed into educational patterns and reasoning notes.

## Rule 4 - Use links aggressively

Every important note should link back to at least one MOC.

## Rule 5 - Convert conversations into durable notes

Whenever a useful discussion happens, convert it into:

1. A permanent note
2. A MOC link
3. Action items
4. Flashcards if useful
5. Product angle if useful
