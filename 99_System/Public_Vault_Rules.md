# Public Vault Rules

This vault is currently designed as a public-safe knowledge vault.

It can contain general medical learning notes, exam frameworks, AI prompts, product ideas, public-safe learning maps, templates, and deidentified educational clinical patterns.

It should not contain anything private, identifying, credential-like, or sensitive while the repository is public.

## Rule 1 - Keep clinical material educational

Clinical notes should be written as patterns, not records.

Use: adult patient with fever and thrombocytopenia - clinical reasoning note.

## Rule 2 - Keep finance notes conceptual

It is okay to learn about debt, cash flow, budgeting, banking, and investing. Keep exact private details outside a public repository.

## Rule 3 - Convert conversations into durable notes

Whenever a useful discussion happens, convert it into one of these forms:

- Permanent note
- MOC page
- Template
- Product idea
- Clinical reasoning framework
- Flashcard source note
- Learning roadmap

## Rule 4 - Link aggressively

Every note should connect to at least one map.
